## 🎯 Project Overview

**TaskIdea** is a cross-platform task management and idea capture application.

**Platforms**: iOS, Android, Web (Browser), Desktop (Electron)  
**Backend**: Google Drive API (no separate backend server)  
**Theme**: Ocean Depths (calming maritime colors)  
**Status**: Development ready

## 🛠️ Tech Stack

### Core
- **Base**: React Native + Expo

## 📊 Key Specifications

### Data Schemas
All data stored as JSON in Google Drive:

**Task**:
```json
{
  "id": "uuid",
  "title": "required string",
  "description": "optional string",
  "dueDate": "YYYY-MM-DD",
  "priority": "high | medium | low",
  "status": "pending | in-progress | completed | blocked",
  "category": "string",
  "tags": ["string"],
  "createdAt": "ISO8601",
  "updatedAt": "ISO8601"
}
```

**Idea**:
```json
{
  "id": "uuid",
  "title": "required string",
  "description": "optional string",
  "topic": "string",
  "tags": ["string"],
  "starred": "boolean",
  "archived": "boolean",
  "createdAt": "ISO8601"
}
```

### Google Drive Structure
```
My Drive/TaskIdea/
├── tasks/
│   ├── tasks_current.json
│   ├── tasks_archive.json
│   └── tasks_metadata.json
├── ideas/
│   ├── ideas_current.json
│   ├── ideas_archive.json
│   └── ideas_metadata.json
└── settings.json
```

## 🔄 Core Features

### Authentication
- Google Sign-In (all platforms)
- OAuth 2.0 flow
- Token management with refresh
- Secure storage per platform

### Task Management
- CRUD operations (Create, Read, Update, Delete)
- Priority levels (High, Medium, Low)
- Status tracking (Pending, In-Progress, Completed, Blocked)
- Due dates and recurring tasks
- Categories and tags
- Search and filtering

### Idea Management
- Quick capture interface
- Organization by topic
- Tagging system
- Star favorite ideas
- Archive functionality
- Search and filtering

### Sync & Offline
- Auto-sync every 30 seconds
- Manual sync option
- Last-write-wins conflict resolution
- Local caching with AsyncStorage
- Offline queue for pending changes
- Background sync on reconnection

## 📱 Platform-Specific Details

### Mobile (iOS/Android)
- Navigation: Bottom tab bar + stack
- Build: Expo EAS Build
- Gestures: Swipe, long-press, pull-to-refresh
- Safe areas for notches

### Web
- Navigation: Responsive drawer + hamburger menu
- Build: Webpack (Expo Web)
- Breakpoints: 375px (mobile), 601px (tablet), 1025px (desktop)
- Storage: IndexedDB for caching

### Desktop (Electron)
- Navigation: Left sidebar + top menu
- Build: Electron Forge
- Features: System tray, keyboard shortcuts, native window
- Storage: Node.js filesystem + keychain

## 🚀 Development Workflow

### Start Development
```bash
npm run mobile:dev    # iOS/Android
npm run web:dev       # Web (port 3000)
npm run desktop:dev   # Electron window
```

### Build for Production
```bash
npm run mobile:build  # iOS/Android
npm run web:build     # Web
npm run desktop:build # Desktop
```

### Code Location
- **Shared code**: `src/` (all platforms use this)
- **Mobile specific**: `mobile/`
- **Web specific**: `web/`
- **Desktop specific**: `desktop/`

## 🔐 Security Requirements

- **Never expose**: API keys, client secrets, tokens in code
- **Storage**: Use `.env` for all credentials
- **Tokens**: Stored securely per platform
- **Data**: Encrypted at rest locally
- **API**: All calls over HTTPS

Project Overview

TaskIdea is a personal productivity app for managing tasks and capturing ideas. It is designed so that all your data lives in your own Google Drive — nothing is stored on any external server. You own your data completely.

FeatureDescription
TasksCreate, complete, and organize tasks with priorities, due dates, and tags
IdeasCapture ideas with color accents, notes, and pin support
SyncAutomatic sync to Google Drive within 1 second of any change
OfflineWorks without internet; changes sync when connection returns
Multi-deviceUse on phone, tablet, and desktop with the same Google account
ThemeOcean Depths — navy, teal, and seafoam dark theme

Google Cloud Console Setup

This is required to enable Google Sign-In and Google Drive access.

1
Create a project Go to console.cloud.google.com → Click "New Project" → Name it "TaskIdea" → Create
2
Enable APIs Go to APIs & Services → Library → Search and enable:
  • Google Drive API
  • Google People API
3
Configure OAuth Consent Screen Go to APIs & Services → OAuth consent screen:
  • User Type: External → Create
  • App name: TaskIdea
  • Support email: your Gmail
  • Scopes: add drive.file, openid, email, profile
  • Publishing status: leave as Testing
4
Add yourself as a Test User On the OAuth consent screen page → click the Audience tab → scroll to "Test users" → Add Users → enter hbudida@gmail.com → Save
5
Create OAuth 2.0 Client IDs Go to APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID:
  • Web: Add Authorized redirect URIs: http://localhost:19006, http://localhost:8081, and your Vercel URL e.g. https://taskidea.vercel.app
  • iOS: Bundle ID = com.taskidea.app
  • Android: Package = com.taskidea.app
6
Copy Client IDs to .env file Open c:\A-Claude\TaskIdea\.env and fill in: 
EXPO_PUBLIC_GOOGLE_CLIENT_ID_WEB=your-web-id.apps.googleusercontent.com
EXPO_PUBLIC_GOOGLE_CLIENT_ID_IOS=your-ios-id.apps.googleusercontent.com
EXPO_PUBLIC_GOOGLE_CLIENT_ID_ANDROID=your-android-id.apps.googleusercontent.com
Important: Never share your .env file or commit it to GitHub. It contains private credentials.

Installing Node.js

Node.js is required to run and build the app.

1
Download Go to nodejs.org → Click the LTS button (left green button) → Download the .msi file
2
Install Run the .msi file → Accept all defaults → Click Install → Allow UAC prompt → Finish
3
Verify Open a new Command Prompt (Win+R → cmd → Enter) and run: 
node --version
npm --version
Both should show version numbers.
Tip: Always use Command Prompt (cmd), not PowerShell, for running npm commands. After installing Node.js, close and reopen your terminal.

Running the App Locally

Open Command Prompt and navigate to the project folder:

cd c:\A-Claude\TaskIdea

First-time setup (run once)

npm install --legacy-peer-deps

Start the web app

npx expo start --web

Opens the app at http://localhost:19006 in your browser.

Start for mobile (Expo Go)

npx expo start

Scan the QR code with the Expo Go app on your phone.

Future updates — redeploy to Vercel

npx expo export --platform web
vercel dist --prod

Deploying to Web (Vercel)

Status: Already deployed at https://taskidea.vercel.app

Steps taken

1
Build web app 
npx expo export --platform web
Creates a dist/ folder with the built web app.
2
Install Vercel CLI and deploy 
npm install -g vercel
vercel dist
3
Add environment variable in Vercel Dashboard vercel.com/dashboard → taskidea project → Settings → Environment Variables → Add: EXPO_PUBLIC_GOOGLE_CLIENT_ID_WEB = your web client ID
4
Add Vercel URL to Google Cloud Console Credentials → Web Client ID → Authorized redirect URIs → add https://taskidea.vercel.app
5
Redeploy with env variable 
npx expo export --platform web
vercel dist --prod

To update the web app after code changes

npx expo export --platform web
vercel dist --prod

Deploying to iOS (Free)

Option A — PWA via Safari Recommended Free

The app is already on Vercel. Just add it to your iPhone home screen:

  1. Open Safari on your iPhone (must be Safari, not Chrome)
  2. Go to https://taskidea.vercel.app
  3. Tap the Share button (box with arrow at the bottom of Safari)
  4. Scroll down and tap "Add to Home Screen"
  5. Tap Add

A TaskIdea icon appears on your home screen. Opens full-screen. Works offline. No expiry.

Option B — Expo Go Free

  1. Install Expo Go from the App Store
  2. Run npx expo start on your PC (same Wi-Fi as iPhone)
  3. Scan the QR code shown in the terminal with your iPhone camera
Limitation: Requires your PC to be on and running. Not suitable for daily standalone use.

Option C — Apple App Store $99/year

Required only if you want to distribute to other users. Not needed for personal use.

Note: Xcode sideloading (another free method) requires a Mac. Since your computer is Windows, this option is not available.

Deploying to Android (Free)

Option A — PWA via Chrome Recommended Free

  1. Open Chrome on your Android phone
  2. Go to https://taskidea.vercel.app
  3. Tap the 3-dot menu (top right)
  4. Tap "Add to Home Screen"
  5. Tap Add

Option B — APK Sideload Free

Builds a real native Android app and installs it directly — no Play Store needed.

1
Create a free Expo account Go to expo.dev → Sign up (free)
2
Install EAS CLI and login 
npm install -g eas-cli
eas login
3
Build the APK 
eas build --platform android --profile preview
Takes 5–10 minutes. Runs on Expo's servers. Gives you a download link when done.
4
Install on Android phone
  • Download the .apk file to your phone
  • Open the file → tap Allow from this source → tap Install

Troubleshooting

Error: redirect_uri_mismatch

The redirect URI in the app doesn't match Google Cloud Console. Go to Credentials → Web Client ID → add http://localhost:19006 and your Vercel URL as Authorized redirect URIs.

Error: Access blocked — app not verified

Go to APIs & Services → OAuth consent screen → Audience tab → Test users → Add Users → enter hbudida@gmail.com → Save.

npm install fails with ERESOLVE

npm install --legacy-peer-deps

The .npmrc file in the project already sets this automatically.

npx / npm not found in PowerShell

Use Command Prompt (cmd) instead of PowerShell. Node.js PATH works correctly in cmd on this machine.

App shows blank screen or crash

  1. Check the terminal for error messages
  2. Make sure .env has correct Google Client IDs
  3. Run npx expo start --web and check the browser console (F12)

Data not syncing

  1. Check internet connection
  2. Pull down to refresh on Tasks or Ideas screen
  3. Go to Settings and tap the refresh icon next to Sync Status
  4. Sign out and sign back in if the issue persists
